# 서이초등학교 방과후 수강신청 시간표 뷰어 기획서

> **최종 업데이트**: 2026-03-08
> **v2 변경 이유**: 실제 사이트 스크린샷(진입화면 + 강좌목록 + 상세정보) 분석으로 초기 추정과 다른 구조 확인. 스크래핑 전략/데이터 모델 전면 수정.

---

## 1. 개요

### 목적
- AfterEdu.kr 서이초등학교 방과후 수강신청 사이트에서 82개 과목 정보를 수집
- 월~금 주간 시간표 형태로 시각화하여 중복 수강 방지

### 문제 정의
- 기존 사이트는 강좌영역별로 과목을 나열하는 방식
- 전체 과목의 요일/시간 충돌을 한눈에 확인할 수 없음
- 중복된 시간대에 여러 과목을 신청하는 실수 발생 가능

## 3. 왜 12개밖에 없었나? (문제 분석)

### 원인 1: 샘플 데이터 = 임시 개발용 목 데이터
- `scraper/data/courses.json`에 직접 작성한 12개 샘플
- 실제 스크래핑 한 번도 실행하지 않은 상태

### 원인 2: 스크래퍼 설계 오류 3가지

**오류 A - 2단계 네비게이션 미처리**
```
❌ 현재 스크래퍼: 랜딩 페이지에서 바로 tr 파싱 시도
✅ 필요: subscribe0.asp → 영역 클릭 → subscribe1.asp 로 이동
```

**오류 B - 아코디언 클릭 없이 파싱 시도**
```
❌ 현재: 페이지 로드 직후 DOM 파싱 (상세정보 미노출 상태)
✅ 필요: 각 과목 행의 ▼ 버튼 클릭 → DOM에 상세정보 노출 후 파싱
```

**오류 C - 한글 요일 파싱 로직 단순화 필요**
```
❌ 현재: "월수목" 문자열 패턴 검색 (실제 형식과 다름)
✅ 필요: "화요일 (13:50 - 14:30), 목요일 ..." 형식 파싱
```

## 5. 업데이트된 스크래핑 전략

### 전체 흐름 (3단계)

```
[Step 1] 진입

  → 리다이렉트 → w2.afteredu.kr
  → 수강신청 버튼 클릭

[Step 2] 강좌영역 순회 (4회 반복)
  subscribe0.asp에서 영역 목록 읽기:
  - "기타영역 (34)"
  - "예술영역 (12)"
  - "체육영역 (25)"
  - "컴퓨터영역 (11)"

  1) 과목 행 목록 수집
  2) 각 과목의 ▼ 아코디언 버튼 클릭
  3) 아코디언 펼쳐진 후 상세 정보 파싱:

  5) 과목 파싱 후 다음 과목으로

[Step 4] 저장
  courses.json 저장 (82개 예상)
```

### 정원/잔여석 파싱
```typescript
// 입력: "0/21명"
function parseCapacity(text: string): { enrolled: number; capacity: number } {
  const match = text.match(/(\d+)\/(\d+)명?/);
  if (!match) return { enrolled: 0, capacity: 0 };
  return { enrolled: parseInt(match[1]), capacity: parseInt(match[2]) };
}
```

### 월수강료 파싱
```typescript
// 입력: "₩25,200원(기수수강료 ₩69,300)"
function parsePrice(text: string): { monthly: number; term?: number } {
  const prices = text.match(/₩?([\d,]+)원?/g) ?? [];
  const nums = prices.map(p => parseInt(p.replace(/[₩,원]/g, "")));
  return { monthly: nums[0] ?? 0, term: nums[1] };
}
```

## 9. 구현 단계 (업데이트)

### Phase 1: 스크래퍼 수정 (최우선)
1. `types/index.ts` Course 인터페이스 업데이트 (grade, classroom, materialCost, termPrice, enrolled, startDate)
2. `scraper/scrape-courses.ts` 전면 재작성
   - subscribe0.asp → 영역 목록 파싱 (4개 영역)
   - 각 영역 클릭 → subscribe1.asp 이동
   - 각 과목 아코디언 클릭 → 상세정보 파싱
   - 한글 요일명 파싱 로직 (`"화요일 (13:50 - 14:30)"` 형식)
   - 월수강료/기수수강료/교재비 분리 파싱
   - 정원 파싱 (`"0/21명"` 형식)
3. 샘플 데이터 `courses.json` → 실제 수집 데이터로 교체
4. 스크래핑 실패 시 오류 구체화 + 재시도 로직

4. `SelectedCourses.tsx` - 교재비 분리 합산 표시
5. 합계 계산 로직 업데이트 (월수강료 + 교재비 분리)

### Phase 3: 사용성 개선 (기존 유지)

3. 데이터 자동 갱신

## 10. 주요 고려사항 (업데이트)

### 스크래핑 제약
- 링크에 사용자 키(`R1176782B0DDF4`)가 포함 → 별도 로그인 불필요
- **✅ 신청기간 외에도 강좌 목록 확인 가능** → 언제든 스크래핑 실행 가능
- 단, 신청기간(`2026-03-09 20:00 ~ 2026-03-10 12:00`) 내에는 잔여석이 실시간 변동
- 아코디언 클릭마다 서버 요청 발생 가능 → 요청 간 `waitForTimeout(500~1000)` 권장
- 82개 과목 × 아코디언 클릭 → 총 실행 시간 약 3~5분 예상
- `headless: true`로 실행 권장 (CI 환경 대응), 디버깅 시 `headless: false`

### 시간표 표시 기준
- 실제 수업 시간: `13:50~14:30`, `15:00~16:00` 등 확인됨
- 시간 범위: 수집된 데이터 기준 자동 산출
- 같은 시간대 복수 과목 → 같은 셀에 카드 나열

### 골프 과목 특이사항 (체육영역)
- 골프(금I), 골프(금J), 골프(목G), 골프(목H) 등 요일+그룹으로 세분화
- 단일 과목이 요일별로 분리된 형태 → 같은 시간이라도 별도 과목으로 처리

### 대상학년 필터 고려
- 1-2학년 / 1-4학년 / 1-6학년 / 2-6학년 / 3-4학년 / 3-6학년 / 5-6학년 등 다양
- 학년 필터: "저학년(1-2)", "중학년(3-4)", "고학년(5-6)", "전학년" 그룹으로 단순화 권장

### 모바일 대응 (변경 없음)
- 데스크탑: 5열 시간표 전체 표시
- 모바일: 요일 탭 전환 방식으로 1일씩 표시
